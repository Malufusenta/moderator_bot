# Chat Moderation Bot

Telegram-бот для модерации объявлений в домовом чате (~700 участников).
Борется со «спящими» участниками: людьми с малой активностью, периодически публикующими объявления о продаже/аренде.

## Сценарий использования

Проект состоит из двух независимых скриптов:

1. **`parser.py`** — запускается **один раз** перед стартом бота. Собирает историю чата через Pyrogram (user-сессию) и заполняет БД историческими данными о активности участников.

2. **`bot.py`** — работает **постоянно**. Модерирует сообщения в реальном времени и предоставляет admin-панель в ЛС.

## Установка

```bash
# 1. Создать виртуальное окружение
python -m venv .venv
source .venv/bin/activate      # Linux/Mac
# или .venv\Scripts\activate   # Windows

# 2. Установить зависимости
pip install -r requirements.txt

# 3. Скопировать и заполнить конфиг
cp .env.example .env
# Открыть .env и заполнить BOT_TOKEN, API_ID, API_HASH, CHAT_ID, ADMIN_IDS
```

## Запуск

```bash
# Шаг 1 — разово, наполнить БД историей (занимает время, делать один раз)
python parser.py

# Шаг 2 — постоянный запуск бота
python bot.py
```

## Конфигурация

Все параметры описаны в `.env.example` с комментариями.
Ключевые пороги модерации:

| Параметр | Значение по умолчанию | Назначение |
|---|---|---|
| `TRUST_LIMIT` | 30 | Минимум сообщений за всё время |
| `RECENCY_DAYS` | 60 | Окно «свежести» активности (дней) |
| `RECENCY_MIN_MESSAGES` | 1 | Минимум сообщений в окне свежести |
| `SIMILARITY_THRESHOLD` | 0.85 | Порог для определения дубля |
| `MUTE_HOURS` | 24 | Длительность мута при нарушении |
| `DRY_RUN` | false | Режим наблюдения: счётчики пишутся, delete/mute/warn — нет |

## Тест-утилиты

### seed.py — подготовка состояний БД для ручного тестирования

```bash
# Засеять «доверенного» участника (count выше порога, был активен сегодня)
python seed.py trusted 123456789 alice

# Засеять «спящего» — высокий count, но последняя активность вышла за окно свежести
python seed.py dormant 123456789 sleeper

# Засеять «новичка» (count=1)
python seed.py newbie 123456789

# Произвольные параметры
python seed.py custom 123456789 --count 15 --days-ago 45 --username bob --ad-attempts 2

# Показать содержимое таблицы users с разбивкой по статусам
python seed.py show

# Очистить все данные (запрашивает подтверждение)
python seed.py reset

# Очистить без подтверждения (для скриптов)
python seed.py reset --yes
```

> **Важно:** `seed.py` использует тот же `DB_PATH`, что и бот.
> Запускайте с тестовой базой (`DB_PATH=data/test.db` в `.env`),
> чтобы не испортить боевые данные.

Все пороги (`TRUST_LIMIT`, `RECENCY_DAYS`) берутся из конфига — поведение
«доверенный»/«спящий» всегда соответствует реальной логике гейта доверия.
